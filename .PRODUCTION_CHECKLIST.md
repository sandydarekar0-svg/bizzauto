# Production Deployment Checklist

## Pre-Deployment Requirements

### Environment Variables
- [ ] Copy `.env.example` to `.env.production`
- [ ] Update `DATABASE_URL` with production database credentials
- [ ] Generate strong `JWT_SECRET` and `JWT_REFRESH_SECRET` (min 32 chars)
- [ ] Update `CORS_ORIGIN` with production domain
- [ ] Set `OPENROUTER_API_KEY` for AI features
- [ ] Configure `META_APP_ID` and `META_APP_SECRET` for WhatsApp
- [ ] Set `RAZORPAY_KEY_ID` and `RAZORPAY_KEY_SECRET` for payments
- [ ] Configure `REDIS_URL` for caching
- [ ] Set up `SMTP_HOST` for email notifications

### Database Setup
- [ ] Create production PostgreSQL database
- [ ] Run `npx prisma generate` to generate Prisma client
- [ ] Run `npx prisma db push` or `npx prisma migrate deploy` to set up schema
- [ ] Verify database connection

### Build Process
- [ ] Run `npm run build` to build both client and server
- [ ] Verify `dist/client` directory exists with frontend build
- [ ] Verify `dist/server` directory exists with compiled server code
- [ ] Test production build locally with `npm run start`

### Security
- [ ] Enable HTTPS in production
- [ ] Set up SSL certificates
- [ ] Configure firewall rules
- [ ] Enable rate limiting (already configured in server)
- [ ] Set up helmet.js headers (already configured)
- [ ] Configure CORS properly for production domain
- [ ] Enable audit logging
- [ ] Set up backup strategy for database

### Performance
- [ ] Enable gzip compression (already configured)
- [ ] Configure CDN for static assets
- [ ] Set up Redis for caching
- [ ] Configure database connection pooling
- [ ] Enable query optimization
- [ ] Set up monitoring and alerting

### Monitoring & Logging
- [ ] Configure Winston logger for production
- [ ] Set up log rotation
- [ ] Configure error tracking (Sentry, etc.)
- [ ] Set up uptime monitoring
- [ ] Configure performance monitoring

### Deployment Options

#### Docker Deployment
- [ ] Build Docker image: `docker build -t your-app-name .`
- [ ] Test Docker container locally
- [ ] Push to container registry
- [ ] Deploy to production server
- [ ] Configure docker-compose for production

#### VPS Deployment
- [ ] Set up VPS (Ubuntu 20.04+ recommended)
- [ ] Install Node.js 20+
- [ ] Install PostgreSQL
- [ ] Install Redis
- [ ] Clone repository
- [ ] Install dependencies: `npm ci --only=production`
- [ ] Build application: `npm run build`
- [ ] Set up PM2 for process management
- [ ] Configure Nginx reverse proxy
- [ ] Set up SSL with Let's Encrypt

#### Cloud Deployment (Vercel/Render/Railway)
- [ ] Connect repository to platform
- [ ] Configure environment variables
- [ ] Set build command: `npm run build`
- [ ] Set start command: `npm run start`
- [ ] Configure database connection
- [ ] Set up custom domain

### Post-Deployment
- [ ] Run smoke tests
- [ ] Verify all API endpoints
- [ ] Test authentication flow
- [ ] Test WhatsApp integration
- [ ] Test payment flow
- [ ] Verify email notifications
- [ ] Set up automated backups
- [ ] Configure disaster recovery plan
- [ ] Document deployment process

### Scaling Considerations
- [ ] Set up load balancer if needed
- [ ] Configure horizontal scaling
- [ ] Set up database read replicas
- [ ] Configure Redis cluster
- [ ] Set up CDN for static assets

## Quick Start Commands

```bash
# Install dependencies
npm ci

# Generate Prisma client
npx prisma generate

# Build application
npm run build

# Start production server
npm run start

# Or with PM2
pm2 start dist/server/index.js --name "bizzauto"
```

## Troubleshooting

### Build Errors
- Check TypeScript compilation errors
- Verify all dependencies are installed
- Check Prisma schema validity

### Runtime Errors
- Check database connection
- Verify environment variables
- Check logs in `error.log` and `combined.log`

### Performance Issues
- Check database query performance
- Verify Redis connection
- Check server resources (CPU, memory)
