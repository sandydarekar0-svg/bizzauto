# Production Deployment Guide

This guide covers deploying the BizzAuto Solutions platform to production.

## Prerequisites

- Node.js 20+ installed
- PostgreSQL database
- Redis server
- Domain name with SSL certificate
- Server with minimum 2GB RAM, 2 CPU cores

## Quick Deployment Options

### Option 1: Docker (Recommended)

```bash
# Clone the repository
git clone <your-repo-url>
cd my-project

# Copy environment file
cp .env.example .env.production

# Edit .env.production with your production values
nano .env.production

# Build and start with Docker Compose
docker-compose up -d

# Check logs
docker-compose logs -f app
```

### Option 2: VPS Deployment

#### 1. Server Setup

```bash
# Update system
sudo apt update && sudo apt upgrade -y

# Install Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Install PostgreSQL
sudo apt install -y postgresql postgresql-contrib

# Install Redis
sudo apt install -y redis-server

# Install Nginx
sudo apt install -y nginx

# Install PM2
sudo npm install -g pm2
```

#### 2. Application Setup

```bash
# Clone repository
git clone <your-repo-url>
cd my-project

# Install dependencies
npm ci --only=production

# Copy environment file
cp .env.example .env.production

# Edit environment variables
nano .env.production
```

#### 3. Database Setup

```bash
# Generate Prisma client
npx prisma generate

# Push schema to database
npx prisma db push

# Or run migrations
npx prisma migrate deploy
```

#### 4. Build Application

```bash
# Build frontend and backend
npm run build
```

#### 5. Start with PM2

```bash
# Start application
pm2 start dist/server/index.js --name "bizzauto"

# Configure PM2 to start on boot
pm2 startup
pm2 save
```

#### 6. Configure Nginx

```nginx
# /etc/nginx/sites-available/bizzauto
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:4000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
# Enable site
sudo ln -s /etc/nginx/sites-available/bizzauto /etc/nginx/sites-enabled/

# Test configuration
sudo nginx -t

# Restart Nginx
sudo systemctl restart nginx
```

#### 7. SSL with Let's Encrypt

```bash
# Install Certbot
sudo apt install -y certbot python3-certbot-nginx

# Obtain SSL certificate
sudo certbot --nginx -d yourdomain.com

# Auto-renewal is configured automatically
```

### Option 3: Cloud Platforms

#### Vercel (Frontend only)

1. Connect your GitHub repository to Vercel
2. Configure build settings:
   - Build Command: `npm run build:client`
   - Output Directory: `dist/client`
3. Add environment variables
4. Deploy

#### Render

1. Create a new Web Service on Render
2. Connect your GitHub repository
3. Configure:
   - Build Command: `npm run build`
   - Start Command: `npm run start`
4. Add environment variables
5. Add PostgreSQL and Redis databases
6. Deploy

#### Railway

1. Create a new project on Railway
2. Add PostgreSQL and Redis services
3. Add your GitHub repository
4. Configure environment variables
5. Deploy

## Environment Variables

Required environment variables for production:

```env
# Server
NODE_ENV=production
PORT=4000

# Database
DATABASE_URL=postgresql://user:password@host:5432/database

# Authentication
JWT_SECRET=your-secret-key-min-32-chars
JWT_REFRESH_SECRET=your-refresh-secret-key-min-32-chars

# CORS
CORS_ORIGIN=https://yourdomain.com

# AI Services
OPENROUTER_API_KEY=your-api-key

# WhatsApp
META_APP_ID=your-app-id
META_APP_SECRET=your-app-secret
WHATSAPP_REDIRECT_URL=https://yourdomain.com/api/whatsapp/callback

# Payments
RAZORPAY_KEY_ID=your-key-id
RAZORPAY_KEY_SECRET=your-key-secret

# Redis
REDIS_URL=redis://localhost:6379

# Email (optional)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password
```

## Monitoring and Maintenance

### View Logs

```bash
# PM2 logs
pm2 logs bizzauto

# Docker logs
docker-compose logs -f app

# Application logs
tail -f combined.log
tail -f error.log
```

### Database Backups

```bash
# Manual backup
pg_dump -U postgres whatsapp_saas > backup_$(date +%Y%m%d).sql

# Automated backup (add to crontab)
0 2 * * * pg_dump -U postgres whatsapp_saas > /backups/backup_$(date +\%Y\%m\%d).sql
```

### Updates

```bash
# Pull latest changes
git pull origin main

# Install dependencies
npm ci --only=production

# Build application
npm run build

# Restart with PM2
pm2 restart bizzauto

# Or with Docker
docker-compose down
docker-compose up -d --build
```

## Troubleshooting

### Build Fails

```bash
# Clear cache
rm -rf node_modules dist
npm install
npm run build
```

### Database Connection Issues

```bash
# Check PostgreSQL status
sudo systemctl status postgresql

# Check connection
psql -U postgres -d whatsapp_saas

# Check Prisma
npx prisma db pull
```

### Port Already in Use

```bash
# Find process using port 4000
lsof -i :4000

# Kill process
kill -9 <PID>
```

### Memory Issues

```bash
# Increase Node.js memory limit
NODE_OPTIONS="--max-old-space-size=4096" npm run start
```

## Security Best Practices

1. **Keep dependencies updated**: Run `npm audit` regularly
2. **Use strong secrets**: Generate random 32+ character strings
3. **Enable HTTPS**: Always use SSL in production
4. **Configure firewall**: Only allow necessary ports
5. **Regular backups**: Automate database backups
6. **Monitor logs**: Set up log aggregation
7. **Rate limiting**: Already configured in the application
8. **Input validation**: All inputs are validated with Zod
9. **SQL injection prevention**: Using Prisma ORM
10. **XSS prevention**: React automatically escapes content

## Scaling

### Horizontal Scaling

```bash
# Start multiple instances with PM2
pm2 start dist/server/index.js -i max --name "bizzauto"

# Or specify number of instances
pm2 start dist/server/index.js -i 4 --name "bizzauto"
```

### Load Balancing

Configure Nginx as a load balancer:

```nginx
upstream bizzauto {
    server localhost:4000;
    server localhost:4001;
    server localhost:4002;
    server localhost:4003;
}

server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://bizzauto;
        # ... other proxy settings
    }
}
```
